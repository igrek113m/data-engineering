## Snowflake, Databricks 紹介

---

### ❄️ Snowflake 紹介

* **① クラウドネイティブなデータ基盤**
   * ストレージとコンピューティングの分離アーキテクチャ
   * マルチクラウド対応（AWS / Azure / GCP）
   * サーバーレス運用による管理コストの低減

* **② データ共有・コラボレーション機能**
   * Snowflake Marketplace によるデータ売買・共有エコシステム
   * Data Clean Room によるプライバシー保護下でのデータ連携
   * 組織をまたいだセキュアなデータ共有（Data Sharing）

* **③ AIとアプリケーション開発への拡張**
   * Cortex AI による LLM 機能のネイティブ統合
   * Snowpark によるPython / Java での処理記述
   * Streamlit 統合によるデータアプリの内製化

---

### 🧱 Databricks 紹介

* **① オープンソース戦略と技術基盤**
   * Apache Spark・Delta Lake・MLflow の開発元としての技術的優位性
   * オープンフォーマット（Delta Lake）によるベンダーロックイン回避
   * Unity Catalog による統合メタデータ・ガバナンス管理

* **② データエンジニアリングと機械学習の統合**
   * Lakehouse アーキテクチャによる DWH と Data Lake の統合
   * MLflow を中心とした ML ライフサイクル管理
   * Feature Store による特徴量の再利用・共有基盤

* **③ 生成AI・LLM開発への対応**
   * DBRX・Dolly などの OSS LLM 開発・公開実績
   * Mosaic AI による LLM のファインチューニング・サービング基盤
   * Vector Search・RAG 構築のためのネイティブサポート

---

> 💡 **発表のポイント**：両社とも「AI への進化」を軸に据えており、Snowflake は **SQL・BIユーザー寄りのアプローチ**、Databricks は **エンジニア・データサイエンティスト寄りのアプローチ** という対比構造で比較すると聴衆に伝わりやすくなります。
